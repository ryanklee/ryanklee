# Ryan Kleeberger

Building Hapax — a single-operator cognitive infrastructure that externalizes executive function for a neurodivergent solo operator. 180 agents, constitutional governance with algebraic proofs, multi-lane AI coordination, multimodal perception, and a 24/7 livestream as research instrument. Running on one workstation, for one person.

## Hapax

| | |
|---|---|
| **What** | Single-operator operating environment: perception, reasoning, action, and self-governance running as one integrated system |
| **Scale** | 180 agents, 326 systemd units, 30,000+ tests, 2,800+ merged PRs, 13 Docker containers, 5 Raspberry Pis |
| **Governance** | 5 constitutional axioms with algebraic verification — consent labels (DLM join-semilattice), provenance (PosBool(X) semiring), deny-wins composition. Extracted as [`agentgov`](https://github.com/ryanklee/hapax-council/tree/main/packages/agentgov) (MIT) |
| **Coordination** | Up to 10 concurrent AI sessions (Claude Code, Codex, Gemini CLI) with relay protocol, triage officer, quota partitioning |
| **Perception** | Voice daemon, 3 USB cameras with GPU compositor, IR fleet, wearable biometrics, 60-node WGSL shader expression engine |
| **Research** | Grounding theory (Clark & Brennan 1991), SCED methodology, refusal-as-data. Cycle 1 pilot complete; Cycle 2 in progress |

### Research framing

> Grounding Attempts as Safety Artifacts: A Live Benchmark for Agentic Claim Confidence, Refusal, and Public Egress.

This is research infrastructure, not a product or community project. The system studies what it means for an agentic system to know what it knows, refuse what it must, and govern what it publishes — under real operational constraints, running 24/7.

Relevant to: empirical AI safety evaluation, agentic oversight, grounded claim authority, privacy-preserving public egress, multi-agent coordination.
